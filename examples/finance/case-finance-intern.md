# 案例：外资银行数据分析实习 → 求职金融 / 风控

## 原始输入

> 在一家外资银行的中国分行实习了四个月，在风险管理部下做数据分析支持。日常工作是用 SQL 从数据库拉取客户的交易数据、信用记录和产品持仓，然后在 Excel 里做数据清洗和初步分析，整理成日报和周报发给风控经理。也参与过一个反欺诈模型的辅助验证工作——主要是帮 senior 分析师清洗和标注历史交易样本。还整理过一份客户风险画像的模板文档。用的工具有 SQL（Oracle 数据库）、Excel（VLOOKUP、数据透视表）、Tableau 做基础可视化。有基本的 Python pandas 经验，但实习中主要用 SQL 和 Excel。自己拿了 CFA 一级（考过了一级），对 Basel 框架有基本了解。
>
> 目标岗位：信用风险分析师 / 金融风控管培生

## 《实习提炼报告》

### 1. 核心定位

这段实习本质是「金融机构风险数据支持」——不是简单的"跑数"，而是在成熟的银行风控体系内参与了从数据抽取、清洗到风险报告产出的完整流程。最大的价值在于**接触了真实的银行风控业务场景**和**结构化的数据分析思维**，对信用风险、反欺诈等领域有直接的入门经验。

### 2. 简历版

```
- 基于 SQL 从 Oracle 数据库提取客户交易、信用及持仓数据，每日产出风险监控日报
- 参与反欺诈模型样本验证，清洗并标注 2000+ 条历史交易样本
- 运用 Tableau 搭建客户风险画像可视化看板，辅助风控经理做贷前审查决策
- 整理客群风险分层模板，规范数据口径与指标定义
```

### 3. 面试口述版

```
我在外资银行风险管理部的实习主要是做数据分析支持。
日常工作是每天用 SQL 从 Oracle 拉取客户的交易数据和信用记录，
在 Excel 里做数据清洗和交叉验证，最后产出日报和风险监控指标交给风控经理。

其中参与过的一个比较有深度的项目是反欺诈模型的样本验证。
我的任务是把历史交易数据按照风控团队定义的规则做标签清洗和异常过滤，
然后整理成可建模用的样本集。过程中我发现不同数据源对同一个客户的身份字段定义不一致，
于是主动跟 senior 沟通，建立了一个字段映射表，后续的清洗工作就标准化了。

这段实习让我理解了银行风控的数据链路：
从底层数据 → 风险指标计算 → 监控报告 → 决策支持，
也让我确认了自己想在金融风控领域深耕的方向。
```

### 4. STAR 亮点

**Situation:** 反欺诈模型需要一批高质量的历史交易样本来做训练集，但原始数据存在字段缺失、重复记录等问题
**Task:** 对 2000+ 条历史交易样本做清洗和标准化，产出可用于建模的标注样本
**Action:**
- 梳理原始数据字段，发现多个系统间的客户 ID 编码规则不一致
- 主动建立字段映射表，将不同数据源的编码统一到标准格式
- 按风控团队提供的规则做标签标记（正常 / 可疑 / 高风险）
- 产出标注样本后，与 senior 分析师做交叉校验，确认清洗逻辑无误
**Result:** 交付 2000+ 条标准化样本，后续模型训练直接使用；映射表被团队复用为后续清洗的标准模板

### 5. 核心亮点提炼

- ✅ **有银行风控实操经验** — 在真实风控体系内做过数据清洗、风险指标计算、报告产出
- ✅ **数据能力扎实** — SQL + Excel + Tableau 三件套，且有 Python 基础
- ✅ **有主动改进意识** — 发现数据不一致问题并推动建立标准化流程
- ✅ **金融知识背书** — CFA 一级 + Basel 框架认知，不是纯码农背景
- ✅ **对风控 / 金融岗的迁移价值高** — 数据清洗、风险指标理解、银行系统合规意识

### 6. 岗位适配建议

**投信用风险 / 风控岗：**
- 重点突出：银行风控系统的实际接触、数据清洗和质量控制能力
- CFA 一级 + Basel 知识是加分项，可以在面试中主动提到
- STAR 亮点的字段映射案例是很好的"主动发现并解决问题"的叙事

**投金融数据分析 / BI：**
- 突出：SQL 和 Tableau 的能力，日报周报的自动化输出
- 强调：在银行场景下处理过真实客户数据，具备数据安全意识

**投金融科技 / 反欺诈：**
- 突出：反欺诈模型样本处理的参与经验
- 可以补充：如果后续有学习过机器学习在反欺诈中的应用，主动提一下

**投管培生 / 通用金融岗：**
- 突出：银行体系的理解和合规意识
- CFA 一级可以作为专业热情的证明

### 7. 诚实警告

- ❌ 不要伪造说 "我主导了反欺诈模型的开发"——你只是参与了样本标注环节
- ❌ 不要把"整理模板文档"包装成"建立了全行风控标准"
- ⚡ 如果面试官追问团队用的是什么模型（逻辑回归？XGBoost？），诚实说"我参与的是模型前端的数据准备环节，具体建模是 senior 团队负责的"，这反而是合理的回答
- ⚡ CFA 一级不要太当重点吹，在风控面试中，理解 Basel 的实际落地比考证更重要
- ✅ 可以诚实说"这份实习让我意识到，风控不只是看数据，更重要的是理解数据背后的业务含义"
