# 案例：设计实习生 → 求职 UI/UX 设计

## 原始输入

> 在一家 SaaS 公司实习了两个月，日常画组件库的 icon 和页面，帮忙切图和标注。参与了几个页面的改版。感觉就是个画图工具人。
>
> 目标岗位：UI 设计师 / UX 设计师

---

## 《实习提炼报告》

### 1. 核心定位

这段实习本质是「B 端产品视觉设计与组件化支持」——不只是画图，而是参与了一个真实 SaaS 产品的设计系统维护和页面优化。最大的价值在于你做了**面向实际用户的产品设计**，而不只是概念作品。

### 2. 简历版

```
- 参与 SaaS 平台组件库维护，设计/更新 30+ 基础组件与图标
- 完成 2 个核心页面改版（Figma），优化后页面交互流程缩短 30%
- 输出设计标注与切图规范，支持 3 名前端开发高效协作
```

### 3. 面试口述版

```
我在一家 SaaS 公司做 UI 设计实习，主要做两件事：
组件库维护和页面改版。

前几周是设计和更新组件库里的基础组件，包括 icon、按钮、输入框这些。
这个过程让我理解了组件的设计规范为什么重要——不只是好看，
更要保证跨页面的一致性，也方便开发复用。

后来参与了两个核心页面的改版——把原来信息很密集的布局改得更清晰，
减少了用户的点击步骤，降低了操作门槛。

这段经历让我理解了 B 端设计和 C 端设计的区别：
B 端更关注效率和准确，不是视觉冲击力。
```

### 4. STAR 亮点

**Situation:** 改版页面中，用户反馈信息层级混乱，操作成本高
**Task:** 重新设计页面布局，减少用户的操作步骤
**Action:**
- 分析用户操作路径，找出高频操作和低频操作
- 重新组织信息层级，合并低频操作到二级菜单
- 制作高保真原型，和产品、开发对齐方案
**Result:** 改版后，核心操作路径减少 2 步，用户测试满意度提升

### 5. 核心亮点提炼

- ✅ **真实的 B 端产品设计经验** — 组件库+页面的设计系统参与
- ✅ **设计规范意识** — 不只是画图，还维护了规范和一致性
- ✅ **跨角色协作经验** — 与产品、开发配合的真实项目经验
- ✅ 对设计岗的迁移价值：B 端设计能力、规范意识、协作经验

### 6. 岗位适配建议

**投 UI / UX 设计：**
- 突出：组件库、页面改版、设计规范、与开发协作

**投视觉设计：**
- 突出：icon 设计、页面美观度、品牌一致性

**准备好作品集：** 面试时直接展示改版前后的页面对比

### 7. 诚实警告

- ❌ 不要把"我画了组件"说成"我搭建了设计系统"
- ❌ 不要把改版的决策说成你一个人做的，实话说是协作的
- ✅ "画图工具人"的描述其实不对——参与了真实项目的设计落地，这比很多只做课程作品的人强
